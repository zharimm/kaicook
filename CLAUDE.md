# kaiCook — Chrome Extension

A Chrome extension that extracts recipes from any webpage using AI and displays them in a clean full-page takeover UI.

## What It Does

1. User visits any recipe webpage and clicks the extension icon
2. Extension captures the current tab's URL and page content
3. AI extracts structured recipe data (ingredients, steps, metadata) and returns JSON
4. A full-page overlay renders the recipe in a clean, readable format
5. User can adjust serving sizes; ingredient quantities scale automatically

## Tech Stack

- **Framework**: [WXT](https://wxt.dev) v0.20.18 — handles manifest generation, HMR, and cross-browser builds
- **UI**: React 19 + TypeScript
- **Build**: ES modules via WXT; `npm run dev` for local dev, `npm run build` for production
- **AI**: Claude API (Anthropic) for recipe extraction — not yet implemented

## Project Structure

```
entrypoints/
  background.ts      # Service worker — coordinates URL capture and AI calls
  content.ts         # Injected into pages — captures DOM content, renders overlay
  popup/             # Extension popup (browser action)
    App.tsx          # Main popup component
    main.tsx         # React entry point
    index.html       # Popup shell
public/
  icon/              # Extension icons: 16, 32, 48, 96, 128px
wxt.config.ts        # WXT configuration
```

## Architecture

### Data Flow

```
User clicks icon
  → popup sends message to background
  → background fetches page content from active tab via content script
  → background calls AI API with page content
  → AI returns structured recipe JSON
  → background sends JSON to content script
  → content script renders full-page overlay UI
```

### Recipe JSON Schema (target)

```typescript
interface Recipe {
  title: string;
  description?: string;
  servings: number;
  prepTime?: string;
  cookTime?: string;
  ingredients: { quantity: number; unit: string; name: string }[];
  steps: string[];
  tags?: string[];
}
```

## MVP Features

- **URL capture**: Detect current tab URL when popup is opened
- **AI extraction**: Send page HTML/text to Claude; parse structured recipe JSON
- **Clean display**: Full-page takeover overlay with recipe rendered in a readable layout
- **Serving size adjustment**: Scale ingredient quantities proportionally when servings change

## Development

```bash
npm install
npm run dev        # Start dev server with HMR; load .output/chrome-mv3 as unpacked extension
npm run build      # Production build
npm run zip        # Package as .zip for Chrome Web Store submission
npm run compile    # TypeScript type check only
```

Load the extension in Chrome: go to `chrome://extensions`, enable Developer Mode, click "Load unpacked", select `.output/chrome-mv3`.

## Key Conventions

- All entrypoints use WXT's `defineBackground` / `defineContentScript` wrappers — do not use raw Chrome APIs without these
- Manifest is auto-generated by WXT from entrypoint file names and metadata; do not create a manual `manifest.json`
- Content script `matches` pattern must be updated from `*://*.google.com/*` to `<all_urls>` to support any recipe site
- The popup (`browser_action`) is the user's entry point; keep it lightweight — delegate heavy work to background
- Use Chrome message passing (`browser.runtime.sendMessage` / `browser.tabs.sendMessage`) for popup ↔ background ↔ content communication
- AI API keys must be stored via `browser.storage.local` or as a build-time env var — never hardcoded

## Current State

The codebase is a WXT + React starter template. All entrypoints contain placeholder/demo code. No AI integration, recipe logic, or overlay UI exists yet. Everything is ready to build on top of.
